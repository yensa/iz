public class ObserverInterconnector

ObserverInterconnector is in charge for inter-connecting subjects with their observers, whatever their specializations are.

With this class, an observer can connect itself to the right subject(s) without having to know it. In the same fashion, the subjects gets their client list automatically filled.

public void beginUpdate()

Several entities will be added. This Avoids any superfluous update while adding. Every beginUpdate() must be followed by an endUpdate(). 

public void endUpdate()

Several subjects or observers have been added. Decrements a counter and update the entities if it's equal to 0.

public void updateObservers()

Updates the connections between the entities stored in the global list.

It has usually not be called manually. During the process, each subject previously added is visited by each observer previously added. The complexity of the operation is usually reduced if beginUpdate() and endUpdate() are used adequatly.
